## 운영체제가 뭐야?

운영체제(Operating System, OS)는 **컴퓨터의 하드웨어 및 여러 시스템 자원들을 관리하는 소프트웨어 입니다.** 우리가 컴퓨터로 웹 브라우저, 게임, 음악 플레이어 등 여러 작업을 수행할 때, 운영체제는 사용자를 대신하여 아래와 같은 작업들을 해 줍니다.

- 마우스, 키보드, 스피커, 모니터 등 입출력 장비들을 관리합니다.
- 실행 중인 프로그램(프로세스)들에 CPU 자원을 배분합니다.
- 주기억장치(RAM)를 관리합니다.
- 컴퓨터 내 수많은 파일들 및 파일 시스템을 관리해 줍니다.
- 인터넷에 접속하고, 외부에서 새 파일을 다운로드 받거나 내부 파일을 외부로 업로드 합니다.
- CD, USB 등 외부 저장 장치들과도 연동합니다.

## 운영체제는 항상 작동하는 소프트웨어!

운영체제 역시 하나의 **소프트웨어**입니다. 하지만 이 소프트웨어는 일반적인 프로그램들과는 달리, **항상 주기억장치에 적재되어 있으며 실시간으로 컴퓨터 시스템을 관리합니다.** 따라서, **운영체제가 멈추면 전체 시스템이 멈춥니다.**(흔히 블루스크린(Blue Screen of Death, BSOD)하면 떠오르는 것이 이 운영체제 문제이며, 후술하겠지만 운영체제가 패닉(Panic) 했다고도 합니다.)

따라서 운영체제는 어떠한 예외 사항이 발생해도, 시스템이 치명적인 위험에 처하지 않도록 가장 견고하고 안전하게 설계되어야 합니다.

### 컴퓨터의 부팅 과정

컴퓨터는 다음과 같은 과정을 통해 부팅(Booting)됩니다.

1. 컴퓨터에 전원이 연결되고, 메인보드, CPU, RAM 등 각종 부품에 전력이 공급됩니다.
2. 메인보드(컴퓨터의 중요 부품들이 모여 있는 하나의 회로 기판.)에 내장된 펌웨어인 바이오스(Basic Input/Output System,BIOS)가 실행됩니다. 바이오스는 POST(Power On-Self Test) 과정으로 컴퓨터의 하드웨어를 점검하며 점검이 끝나면 CMOS 설정에 따라 부팅 가능한 장치들(하드디스크, SSD, USB 메모리 등)을 찾습니다.(최근 컴퓨터들은 바이오스 대신 성능이 개선된 UEFI 방식을 주로 사용하지만, 기본적인 역할은 비슷합니다.)
3. 바이오스가 부팅 장치의 첫 번째 섹터에 있는 MBR에 접근해 부트 로더(Boot Loader)를 찾아 메모리에 적재합니다. 부트 로더는 이름 그대로 컴퓨터 부팅 과정만을 위해 실행되는 특별한 프로그램입니다.
4. 부트 로더가 실행되면 운영체제의 핵심인 커널(Kernel)을 메모리에 적재하고 커널 실행에 필요한 환경을 설정합니다.
5. 메모리에 적재된 커널을 실행합니다.
6. 시스템 서비스, 사용자 로그인 환경 구성, 네트워크 설정 등 다양한 설정을 합니다.
7. 사용자가 컴퓨터에 로그인합니다.

여기서 3번과 4번 과정을 눈여겨보시기 바랍니다. 부트 로더가 실행된 직후, 우리가 앞으로 다룰 운영체제(커널)가 메모리에 올라가는 것을 볼 수 있습니다. 이처럼 운영체제는 다른 어떤 프로그램보다 가장 먼저 메모리에 자리 잡으며, 컴퓨터가 꺼질 때까지 전체 시스템을 든든하게 관리합니다.

## Linux 운영체제의 구조

리눅스 운영체제는 역할과 권한에 따라 크게 다음과 같은 요소들로 구성되어 있습니다.

1. **커널 (Kernel)**
   * **위치:** 커널 영역 (Kernel Space / Ring 0)
   * 운영체제의 심장이자 핵심입니다. CPU, 메모리, 디스크, 네트워크 등 실제 하드웨어 자원을 직접 통제하고 관리하는 전지전능한 프로그램입니다.

2. **시스템 라이브러리 (System Libraries)**
   * **위치:** 사용자 영역 (User Space / Ring 3)
   * 일반 응용 프로그램이 커널과 안전하게 소통할 수 있도록 인터페이스(함수)를 제공합니다. 가장 대표적인 것이 바로 C 표준 라이브러리인 `libc`입니다. 우리가 자주 쓰는 `open()`, `read()` 같은 함수들도 결국 내부적으로 커널에게 작업을 부탁하는 시스템 호출(System Call)을 감싸고 있습니다.

3. **시스템 관리 프로그램 (Utility & System Daemons)**
   * **위치:** 사용자 영역 (User Space / Ring 3)
   * OS가 정상적으로 돌아가도록 뒷단에서 특수한 관리 임무를 수행하는 프로그램들입니다. 네트워크 데몬, 로그인 처리 프로세스, 시스템 로그 관리자 등이 여기에 속합니다.

4. **사용자 영역 (User-Land / User Space)**
   * **위치:** 사용자 영역 (User Space / Ring 3)
   * 운영체제의 핵심부 외에, 사용자가 직접 실행하는 모든 공간을 의미합니다. 웹 브라우저, 게임, 컴파일러(`gcc`), 우리가 작성한 C/Python 코드, 그리고 bash 같은 셸 프로그램이 모두 이 영역의 메모리 위에서 제한된 권한으로 동작합니다.

Linux 운영체제는 여기서 1, 2, 3 번 영역에 해당합니다. 4번은 컴퓨터에서 우리가 주로 사용하는 영역이랍니다!

위 영역에서 가장 중요한 것은 1번, 즉 커널입니다. 리눅스를 포함하여 많은 운영체제들은 메모리 상에서 두 영역(권한)으로 나뉩니다.

- 커널 영역(Kernel Space) : 커널이 위치한 곳으로, 컴퓨터의 모든 자원에 대한 접근 권한을 갖습니다.
- 사용자 영역(User Space) : 사용자가 사용하는 곳으로, 시스템 자원들 중 통제된 일부만 사용할 수 있습니다.

일반적으로 사용자는 작업을 하면서 커널 영역에 임의로 접근할 수 없습니다. 따라서 사용자는 프로그램을 실행할 때 **커널에 특정한 작업을 요청(입/출력, 연결 등)을 수행하도록 요청**해야 합니다. 이것을 시스템 호출(System Call)이라 부릅니다.

사용자가 시스템 호출을 요청하면, 사용자 프로그램이 직접 커널 권한을 가지는 것이 아닙니다. 대신, 권한을 가진 커널이 사용자의 부탁을 받아 잠시 넘겨받아 안전하게 작업을 대행해 주고, 처리가 끝나면 그 결과와 함께 다시 사용자 영역으로 돌아와 이후 작업을 이어간답니다! (이처럼 유저 영역과 커널 영역을 넘나드는 과정을 **문맥 전환(Context Switching)**이라고 부릅니다.)

C언어에서 printf, fopen 같은 함수들도 내부 구현상으로는 시스템 호출을 사용하며, 우리가 사용하는 거의 모든 소프트웨어들도 필요에 따라 시스템 호출을 합니다.(리눅스 터미널에서 strace 를 사용하면 프로그램이 실행 중 어떤 시스템 호출을 하는지 추적할 수 있습니다.)
