# 0-2. Linux 커널 구조 이해하기

## 운영체제의 구조

현대 운영체제들은 여러 기능이 결합된 거대한 유기체입니다. 소프트웨어가 거대해질수록 이를 유지/보수하기 편하게 하려면, 신중하게 제작되어야 합니다!

일반적으로 커널은

|구조|설명|
|---|---|
|모놀리틱(Monolithic) 구조|운영체제의 모든 기능이 하나의 프로그램 안에 구현되어 있습니다.<br>가장 단순한 구조이며, 모든 서비스가 같은 메모리 영역에서 실행되므로 성능이 가장 좋습니다.<br>하지만 커널이 지나치게 거대해지면서 수정 및 확장이 힘들어지고, 커널 영역 한 곳에서 오류가 발생하면 커널 전체가 마비됩니다.|
|마이크로커널(Microkernel)|커널 영역에는 메모리 관리, CPU 스케줄링 등 최소한의 필수 기능만 남기고, 파일 시스템, 장치 드라이버 등은 사용자 영역으로 분리한 구조입니다.<br>서비스의 대부분이 사용자 영역에서 수행되므로, 한 드라이버에서 오류가 발생해도 운영체제의 다른 부분이 영향을 받지 않습니다.<br>하지만 구조상 사용자 영역과 커널 영역을 자주 왕래해야 하므로, 모놀리식 구조에 비해 성능이 떨어집니다.|
|적재가능 커널 모듈(Loadable Kernel Modules, LKM)|커널은 핵심 기능을 제공하며, 다른 커널 기능 및 서비스들을 커널이 실행되는 동안 원하는 대로 시스템에 적재(Mound)/호출 될 수 있습니다.|

Linux 운영체제의 창시자인 리누스 토르발즈는 **모놀리틱 구조**를 채택했습니다. 하지만 모놀리틱 구조는 새로운 하드웨어 및 기능이 나올 때마다 커널 전체를 다시 컴파일 해야 하는 불편함이 있었고, 이를 해결하기 위해 적재가능 커널 모듈(LKM)을 도입했습니다. 기본 구조는 모놀리틱 이지만, **필요할 때마다 드라이버나 파일 시스템을 모듈의 형태로 커널 영역에 적재**하는 방식을 취했습니다.

Linux는 오픈 소스 프로젝트이므로, 현재도 세계의 수많은 개발자들이 새로운 모듈을 만들고 이를 공유하고 있습니다. 공식 운영체제(메인라인) 영역에 포함되는 모듈들은 엄격한 코드 리뷰를 거친답니다!
